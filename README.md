# ✨ SupportFlow AI — Интеллектуальный HelpDesk & IT-Support

**SupportFlow AI** — это современная система управления заявками (Ticketing System), которая использует возможности искусственного интеллекта для автоматизации работы первой линии поддержки. Система автоматически анализирует поступающие запросы, определяет категорию проблемы, присваивает приоритет срочности (SLA) и даже генерирует черновики ответов или первичные рекомендации для инженеров.

---

## 🎯 Проблема и Решение (Зачем нужен этот проект? — Для защиты)

**Проблема:**
В средних и крупных компаниях IT-отделы (первая линия поддержки) тратят до 40% своего рабочего времени на рутинную классификацию заявок. Определение того, к кому отправить тикет (сетевикам, сисадминам, безопасникам), и какой приоритет ему выставить, отнимает время, критичное для срочных проблем (P1/High). Из-за этого нарушаются SLA (Service Level Agreement), а пользователи долго ждут ответа.

**Решение — SupportFlow AI:**
Проект делегирует первичную сортировку искусственному интеллекту (Google Gemini API). Как только пользователь описывает свою проблему, LLM-модель:
1. Вычленяет суть и назначает правильный **Приоритет** (Low, Medium, High).
2. Классифицирует по **Категории** (Network, Hardware, Access Rights, Security и др.).
3. Дает **AI-подсказку** — либо готовое решение, либо наводящие вопросы для пользователя.

**Бизнес-ценность:**
* Снижение **MTTR** (Mean Time To Resolve) за счет мгновенной маршрутизации.
* Автоматический контроль рисков нарушения SLA (цветовая индикация и аналитика).
* Снижение нагрузки и влияния человеческого фактора на 1-й линии поддержки.

## 🚀 Основной функционал (Features)

*   **🧠 AI Auto-Tagging & Response:** Встроенная интеграция с Gemini API. Интеллектуальный анализ описания заявки "на лету" с авто-заполнением мета-данных и генерацией ответа (`AI Processed Response`).
*   **⏱️ Продвинутый контроль SLA:** 
    * Динамический расчет времени отклика в зависимости от приоритета (High - 15 мин, Medium - 60 мин, Low - 24ч).
    * Подсветка просроченных заявок (как в списках, так и на дашборде).
*   **📊 Аналитический Дашборд (Real-time Analytics):**
    * Визуализация количества заявок по категориям и приоритетам.
    * Индикаторы производительности IT-отдела: **SLA On-Time Percentage** (процент заявок, закрытых в срок) и Среднее время обработки.
    * Блок "Динамика по дням" и мониторинг "Загруженности системы" в реальном времени.
*   **🎛️ Умный интерфейс (UI/UX):**
    * Полностью адаптивный дизайн (Desktop + Mobile). Mobile-first подход к дашборду и таблицам.
    * Темная техническая тема (Dark Mode), разработанная для комфорта глаз инженеров поддержки.
    * Сворачиваемый сайдбар (Collapsible Sidebar) для увеличения рабочей области на широких мониторах.
    * Удобные фильтры и встроенный поиск.

## 🛠️ Используемые технологии (Tech Stack)

Приложение построено с использованием современного стека фронтенд-технологий:

*   **Frontend Framework:** React 18 + TypeScript (строгая типизация данных заявок и API ответов).
*   **Сборка:** Vite (высокая скорость сборки и HMR в разработке).
*   **Стилизация:** Tailwind CSS (utility-first подход для быстрого и единообразного дизайна, кастомные анимации `animate-fade-in`, адаптивная верстка).
*   **Роутинг:** React Router v6.
*   **Визуализация данных:** Recharts (отзывчивые Pie, Bar и Line графики на дашборде).
*   **Иконки & Анимации:** Lucide React, Motion (framer-motion).
*   **AI Integration:** `@google/genai` (Официальный SDK для взаимодействия с Google Gemini API).

## ⚙️ Архитектура и "Как это работает"

1.  **Создание заявки (Create Ticket):**
    Пользователь заполняет форму (или выбирает один из частых шаблонов). На этом этапе фронтенд отправляет запрос к сервису `ai.ts`, который формирует Structured Prompt для Gemini и просит вернуть JSON со следующими полями: `priority`, `category`, `ai_response`.
2.  **Управление списком (Ticket List):**
    Запросы рендерятся в адаптивной таблице. Компонент `TicketCard` "на лету" высчитывает время существования тикета и сравнивает его с внутренними лимитами SLA. Если дедлайн прошел, включается триггер `isOverdue` и тикет подсвечивается красным.
3.  **Аналитика (Dashboard):**
    Использует `useMemo` для тяжелых вычислений агрегированной статистики (Total, SLA metrics, Grouping by categories) на стороне клиента, после чего данные передаются напрямую в конфигурации компонент Recharts для отрисовки графиков. Также на дашборде вызывается AI для генерации текстового инсайта (сводки) по текущей ситуации.

## 🏃‍♂️ Как запустить локально

1. Склонируйте репозиторий:
   ```bash
   git clone <url_репозитория>
   cd support-flow
   ```
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Добавьте ваш ключ Google Gemini API в переменные окружения:
   Создайте файл `.env.local` в корне проекта и добавьте:
   ```
   GEMINI_API_KEY=ваш_api_ключ
   ```
4. Запустите dev-сервер:
   ```bash
   npm run dev
   ```
5. Откройте `http://localhost:3000` в вашем браузере.
